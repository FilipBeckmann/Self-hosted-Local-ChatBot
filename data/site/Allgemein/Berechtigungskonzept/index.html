    Berechtigungskonzept

    In Kosy Enterprise ist ein Berechtigungskonzept hinterlegt. Das bedeutet, dass Rechte auf Programmfunktionen vergeben werden können, so dass Tätigkeiten wie die Bearbeitung von Strukturen, das Aktualisieren von Werten und die Erstellung von Berichten nur für die jeweils zuständigen Mitarbeiter freigeschaltet werden können.

    Die möglichen Berechtigungsarten sind:

    • kein Zugriff
    • Lesezugriff
    • Änderungszugriff
    • Schreibzugriff
    • Vollzugriff

    Die Rechtevergabe kann nicht nur auf den Funktionen von Kosy Enterprise erfolgen, sondern auch auf inhaltlicher bzw. struktureller Ebene. So können Sie beispielsweise Kostenstellen oder Konten nur für ausgewählte Mitarbeiter bereitstellen.

    Der Administrator besitzt im gesamten Programm Vollzugriff. Die verschiedenen Rechtevergaben dieses Menüpunktes werden im Folgenden erläutert.

    Bedeutung der einzelnen Berechtigungsarten

    kein Zugriff

    Besitzt ein Benutzer keine Zugriffsrechte, so ist es ihm nicht möglich, sich in Kosy Enterprise anzumelden. Jeglicher Zugriff wird ihm somit verwehrt. Wird eine Anmeldung getätigt, so erscheint eine Fehlermeldung.

    Bezieht sich die Berechtigungsart kein Zugriff auf Funktionen oder Elemente, so werden dem Benutzer die jeweiligen Funktionen oder Elemente ausgegraut bzw. als unkenntlich dargestellt.

    Lesezugriff

    Ein Lesezugriff ermöglicht dem Benutzer, sich Funktionen, Elemente etc. anzuschauen. Er besitzt jedoch nicht die Möglichkeit, etwas zu verändern oder Neues anzulegen. Grafisch erkennen Sie diese Einstellung entweder an dem Symbol des Buches oder an dem Kreis hinter dem Funktionsnamen in der Menüleiste.

    Änderungszugriff

    Ist der Änderungszugriff aktiviert, so befähigt das den jeweiligen Benutzer dazu, Elemente zu ändern (z. B. Konten umzubenennen). Es ist ihm jedoch nicht erlaubt, neue Elemente anzulegen. Grafisch erkennen Sie diese Einstellung entweder an dem Symbol der Schalthebel oder an den doppelten Anführungszeichen.

    Schreibzugriff

    Der Schreibzugriff autorisiert den Benutzer dazu, Elemente zu ändern und neue Elemente anzulegen. Ein Löschen von Elementen ist ihm untersagt. Grafisch erkennen Sie diese Einstellung entweder an dem Symbol des Bleistiftes oder an dem einfachen Anführungszeichen.

    Vollzugriff

    Durch den Vollzugriff erlangt der Benutzer für die jeweiligen Funktionen oder Elemente die gleichen Rechte wie ein Administrator. Somit ist er befähigt, bestehende Elemente zu lesen, zu ändern, zu löschen sowie neue Elemente anzulegen. Grafisch erkennen Sie diese Einstellung an dem Symbol der Krone .

    Hierarchie der Rechte

    Sie können in Kosy Enterprise an verschiedenen Stellen Rechte vergeben. Hierbei werden unterschiedliche Rechtehierarchien angewendet. Für die erste Benutzung von Kosy Enterprise ist es wichtig, dass Sie als Administrator die jeweiligen Benutzer anlegen und Ihnen in der Benutzerübersicht (siehe Abschnitt Benutzerübersicht) den gewünschten Zugriff erteilen. Des Weiteren können Sie Rechte auf Module, Funktionen, Mandanten und Dateien erteilen. Sie haben sogar die Möglichkeit, Ihre Rechtevergabe bis auf Menüpunktebene zu erstrecken. Bei diesen Kombinationen gilt immer das niedrigste Recht des Benutzers.

    Beispiel

    Wurde einem Benutzer in der Benutzerübersicht das Recht des Lesezugriffs gegeben, ihm jedoch bei verschiedenen Funktionen (unter Verwaltung > Rechte > Rechte auf Funktionen) ein Schreibzugriff erteilt, so wirkt der Lesezugriff, da es das niedrigere Recht ist.

    Des Weiteren wird in Kosy Enterprise eine Trennung von Benutzer- sowie Gruppenrechten vorgenommen. Benutzerrechte beziehen sich auf eine einzelne Person. Durch die Vergabe von Gruppenrechten ist es Ihnen möglich, mehrere Benutzer zu einer Gruppe zusammenzufassen, um diesen einheitliche Rechte zu geben. Überschneiden sich hier mögliche Rechte, so gilt das höhere Recht.

    Beispiel

    Ein *Benutzer X hat das Recht alle Mandanten zu sehen (Verwaltung > Rechte > Rechte auf Mandanten > Benutzerrechte auf Mandanten). Er besitzt also einen Lesezugriff. Weiterhin besteht eine Gruppe, die die Mandanten im System pflegen soll. Diese Gruppe hat einen Schreibzugriff. Sie darf also bestehende Mandanten ändern sowie neue Mandanten anlegen. Der Benutzer X gehört dieser Gruppe an und darf somit trotz seines generellen Lesezugriffs auf Mandaten in diesem Fall auch schreiben. Das höhere Recht zählt.*

    Sie finden die Unterscheidung von Benutzer- und Gruppenrechten in den jeweiligen Rechtefunktionen am linken Rand des Bildschirms in den zwei unterschiedlichen Registerkarten.

    Einfache Listenansicht

    In der einfachen Listenansicht ordnet man dem Benutzer die jeweiligen Rechte zu den aufgelisteten Modulen durch Markieren dieser zu. Alternativ können Sie die Dimensionen vertauschen , so dass das Modul statt des Benutzers in der Auswahlbox erscheint. Über die Symbole in der Bearbeitungsleiste haben Sie die Möglichkeit, dem Benutzer über die Weltkugel alle Module mit dem ausgewählten Recht zuzuordnen bzw. über die negierte Weltkugel alle Zuordnungen aufzuheben, Zuordnungen umzukehren und nach Modulen zu suchen.

    Das ausgewählte Recht finden Sie ganz links in der Bearbeitungsleiste. Durch einen Linksklick auf das Symbol, können Sie zwischen den einzelnen Berechtigungen wechseln, alternativ ist dies auch durch die unten aufgeführten Tastenkombinationen möglich. Nach Anwahl des gewünschten Rechtes klicken Sie bitte auf ein Modul, um dieser das entsprechende Recht zu geben. Sie erkennen das gesetzte Recht an dem Buchstaben vor dem jeweiligen Modul. Bei erneutem Klicken auf das Modul wird das vorher gesetzte Recht entfernt. Der Benutzer besitzt nun keinen Zugriff auf dieses Modul.

    Symbol Bezeichnung Tastenkombination Funktionsweise
    V Vollzugriff Strg+V Der Benutzer/die Benutzer bekommt/en für das markierte Modul einen Vollzugriff.
    S Schreibzugriff Strg+S Der Benutzer/die Benutzer bekommt/en für das markierte Modul einen Schreibzugriff.
    Ä Änderungszugriff Strg+Ä Der Benutzer/die Benutzer bekommt/en für das markierte Modul einen Änderungszugriff.
    L Lesezugriff Strg+L Der Benutzer/die Benutzer bekommt/en für das markierte Modul einen Lesezugriff.

    Symbolerläuterung Listenansicht

    Hinweis

    Möchten Sie einem einzelnen Modul ein bestimmtes Recht geben, so wählen Sie dieses Modul bitte durch einen einfachen Linksklick aus. Nun können Sie über die Tasten 0, 1, 2, 3 und 4 direkt das jeweilige Recht vergeben (zur Erläuterung der Tasten s. Tabelle "Berechtigungsvergabe über die Tastatureingabe".)

    Haben Sie die Dimensionen vertauscht, so gelten die Symbole in der Zuordnung analog für die Benutzer. Alternativ können Sie per Rechtsklick das Kontextmenü mit den entsprechenden Funktionen aufrufen.

    Nach Ihrer getroffenen Zuordnung speichern Sie Ihre Wahl über oder verwerfen diese über .

    Natürlich können Sie auch hier wieder über Doppelklick auf die Leiste im unteren Bildschirmrand das Feld zur Notizeingabe einblenden.

    Geteilte Listenansicht

    Die geteilte Listenansicht besteht aus dem Benutzer in der Auswahlbox und zwei Ansichten der Module. In der linken sowie rechten Fensterhälfte befinden sich Registerkarten mit den Berechtigungen und den darin bereits zugeordneten Modulen. Markieren Sie auf der linken Seite ein oder mehrere Module, so können Sie diese Auswahl mit dem Pfeil nach rechts in die rechte Fensterhälfte verschieben. Somit haben Sie dem Modul für diesen Benutzer das entsprechende Recht zugeordnet.

    Durch das weitere Symbol mit dem Pfeil nach rechts haben Sie die Möglichkeit, alle Module der linken Fensterhälfte direkt nach rechts zu verschieben. Analog können Sie natürlich auch Zugehörigkeiten wieder aufheben, indem Sie bestimmte Module mit dem Pfeil nach links wieder in die linke Hälfte zurückschieben.

    Nach Ihrer getroffenen Zuordnung, speichern Sie Ihre Wahl über oder verwerfen diese über . Auch hier ist ein Wechsel der Dimensionen möglich , damit das Modul über die Auswahlbox anwählbar ist.

    Makros

    Über Makros haben Sie die Möglichkeit, durch reguläre Ausdrücke Zuordnungen durchzuführen. Legen Sie hierfür ein neues Makro über den Button an. Die Kurzbezeichnung dient hier wieder der eindeutigen Markierung des Datensatzes. Der Titel ist frei wählbar, sollte jedoch sprechend sein. In dem Feld Benutzer-Auswahl können Sie nun über einen regulären Ausdruck mehrere Benutzer ansprechen, die Sie dann einem Modul, ebenfalls definiert durch einen regulären Ausdruck, zuordnen (Modul-Auswahl). Natürlich ist auch hier eine Umkehrung möglich, so dass Sie Modulen Benutzern zuordnen. Dies können Sie im Feld Richtung einstellen. Im Feld Art definieren Sie pro Makro, welche Berechtigung der gewählten Benutzer-Modul-Kombination hinterlegt werden soll.

    Zusätzlich zu der Benutzer- und Modul-Auswahl haben Sie die Möglichkeit, über Benutzer-Einschränkung die Makros gezielt auf einzelne Gruppen ausführen zu lassen. Hierzu wählen Sie in der Spalte Benutzer-Einschränkung bitte die jeweilige Gruppe aus. Damit wird das Makro ausschließlich auf diese Gruppe angewendet.

    Lassen Sie Ihre Makros automatisch pflegen. Standardmäßig ist hier die Auswahl „immer" getroffen. Damit überprüft Kosy Enterprise stets, ob alle Elemente, die von einem Makro angesprochen werden, eine Berechtigung erhalten haben (auch neu angelegte Elemente!). Dadurch ist eine erneute Ausführung eines Makros nicht mehr notwendig.

    Nach Eingabe des Makros führen Sie bitte dieses über die Startflagge aus. Alternativ können Sie auch einen Klick auf den Haken neben der Flagge machen. Es öffnet sich ein Kontextmenü mit weiteren Möglichkeiten. Sind mehrere Makros vorhanden, so können Sie diese alle gleichzeitig ausführen. Nun können Sie das Ergebnis z. B. in der Matrixansicht überprüfen.

    Gruppenrechte

    In Kosy Enterprise ist es zusätzlich möglich, Benutzergruppen bestimmte Rechte zuzuordnen. Das Vorgehen unterscheidet sich hierbei nicht von der Vergabe von Benutzerrechten (siehe Abschnitt Benutzerrechte).
